## How It Works

### For Students

**1. Sign Up & Deposit**
```
User deposits $10 → Receives 10,000 ADC (Academix Digital Currency)
Conversion: 1 USD = 1,000 ADC
```

**2. Choose Game Mode**

**1v1 Mode (Quick Match):**
- 2 players compete head-to-head
- 3-8 questions
- Entry: 300-800 ADC ($0.30-$0.80)
- Winner takes 75%, Loser gets 20%
- Duration: 2-5 minutes

**Multiplayer Mode (Tournament):**
- 6-100 players compete
- 10-20 questions
- Entry: 700-1,400 ADC ($0.70-$1.40)
- Top 30% win prizes (weighted distribution)
- Duration: 10-30 minutes

**3. Play & Earn**
- Answer questions correctly
- Faster answers = more points
- Top performers win ADC
- Even losers get consolation prizes

**4. Withdraw Anytime**
- Minimum: 5,000 ADC ($5)
- Withdrawal fee: 2% + $0.50
- Arrives in 1-24 hours via Flutterwave
- Methods: Bank transfer, mobile money, card

### For Educators (Creators)

**1. Create Quiz**
- Upload 10-20 questions
- Add images, explanations
- Set difficulty level
- Choose subject/category

**2. AI Pre-Screening**
- AI checks factual accuracy
- Grammar/spelling corrections
- **Takes 30 seconds**

**3. Human Review**
- Reviewer verifies quality
- Approves or requests changes
- Approval time: 1-24 hours

**4. Earn Per Play**
- 1v1 Mode: $0.02-$0.08 per question
- Multiplayer: $0.05-$2.80 per quiz
- Scales with popularity
- Passive income (quiz plays forever)

**Example Earnings:**
- Quiz with 10 questions
- Played 100 times/month
- Earnings: $50-280/month per quiz
- Create 10 popular quizzes = $500-2,800/month

### For Reviewers

**1. Review Quizzes**
- Verify accuracy
- Approve/reject
- Takes 5-10 minutes per quiz

**2. Earn Per Review**
- Similar to creators: $0.02-$0.08 per question
- Can review 50-100 quizzes/day
- Earnings: $100-800/day (full-time)

## Business Model

### Virtual Currency System (ADC)

**Why Virtual Currency?**
1. **Regulatory advantage** - Not classified as gambling in most jurisdictions
2. **Better UX** - "10,000 ADC" feels less scary than "$10"
3. **Lower transaction costs** - Only pay fees on deposit/withdrawal, not every game
4. **Float revenue** - Earn interest on unredeemed balances
5. **Proven model** - Roblox ($3B revenue), Fortnite ($5B revenue)

**Conversion Rates:**
- 1 USD = 1,000 ADC
- 1 NGN (Nigeria) = 1.5 ADC
- 1 KES (Kenya) = 7.5 ADC
- 1 INR (India) = 12 ADC

**Transaction Flow:**
```
Deposit: User → Flutterwave (3% fee) → Academix → ADC Wallet
Gameplay: ADC Wallet → Quiz Entry → Winners/Losers
Withdrawal: ADC Wallet → Academix (2% fee) → Flutterwave → User Bank
```

### Payout Algorithm

**1v1 Mode:**
```
Total Pool = 2 × Entry Fee
Winner = 75% of pool
Loser = 20% of pool
Platform = 5% of pool
Creator/Reviewer = Split from platform fee
```

**Multiplayer Mode (Intelligent Distribution):**
```
Total Pool = N players × Entry Fee
Development Charge = 2.38-20% (scales with player count)
Remaining Pool = Distributed to winners

Winner Calculation:
- Winners = ceil((3 × N) / max(min_players, 10))
- Minimum 3 winners, maximum N winners
- Top tier: 40% of prize pool
- Mid tier: 35% of prize pool
- Bottom tier: 25% of prize pool

Non-Winners (Ranked Rewards):
- Weighted distribution based on rank
- Higher ranks get more
- Everyone gets something (retention)
```

**Example (20 players, 980 ADC entry):**
```
Total Pool: 19,600 ADC ($19.60)
Dev Charge (20%): 3,920 ADC
Prize Pool: 15,680 ADC

Winners (6 players):
- Rank 1-2: 3,920 ADC each ($3.92) - 300% ROI
- Rank 3-4: 3,360 ADC each ($3.36) - 243% ROI
- Rank 5-6: 2,800 ADC each ($2.80) - 186% ROI

Ranked (14 players):
- Rank 7: 728 ADC ($0.73)
- Rank 8: 672 ADC ($0.67)
- ...
- Rank 20: 52 ADC ($0.05)

Creator: 980 ADC ($0.98)
Reviewer: 980 ADC ($0.98)
Platform: 1,960 ADC ($1.96)
```

### 3. Intelligent Question Tracker System ✅

**Revolutionary Learning Algorithm:**

**How It Works:**
```
For Each Player:
- Track: Correct answers, Missed answers, Never attempted
- Only show: Missed + Never attempted questions
- Never repeat: Questions answered correctly

When Players Join Quiz:
1. Each player brings their pool of (Missed + Never attempted)
2. System creates intersection of all players' pools
3. Randomly selects required questions from combined pool
4. Shuffles question order per player (prevents cheating)
```

**Example (3 players, 10 questions needed):**
```
Player A pool: 500 questions (missed/new)
Player B pool: 450 questions (missed/new)
Player C pool: 480 questions (missed/new)

Combined pool: ~400 questions (intersection)
System selects: 10 random questions

Player A sees: Q3, Q7, Q1, Q9, Q2, Q5, Q8, Q4, Q10, Q6
Player B sees: Q8, Q2, Q5, Q1, Q9, Q3, Q6, Q10, Q4, Q7
Player C sees: Q5, Q9, Q3, Q7, Q2, Q8, Q1, Q6, Q4, Q10

Same questions, different order = Fair + Anti-cheat
```

**Why This is GENIUS:**

1. **Adaptive Learning** ✅
   - Students only see questions they need to learn
   - No wasted time on mastered content
   - Personalized difficulty per player

2. **Spaced Repetition** ✅
   - Missed questions reappear until mastered
   - Proven learning technique (Anki, SuperMemo)
   - 2-3× faster learning vs. random questions

3. **Anti-Cheating** ✅
   - Each player sees different question order
   - Can't copy neighbor's answers
   - Can't collude ("Answer B for Q1")

4. **Fair Competition** ✅
   - All players see questions at their level
   - No advantage from seeing easy questions
   - Skill-based, not luck-based

5. **Infinite Content** ✅
   - 10,000 questions in database
   - Each player sees unique subset
   - Never runs out of content

6. **Data Goldmine** ✅
   - Track which questions are hardest
   - Identify knowledge gaps
   - Improve content quality
   - Personalized recommendations

**Competitors Have NOTHING Like This:**
- Kahoot: Random questions, everyone sees same order
- Quizizz: Random questions, no tracking
- Duolingo: Linear progression, not competitive

**This is a 10× Better Learning Experience.**

**Patent Potential:** This algorithm could be patentable as a "Method for Adaptive Competitive Learning"

## Implementation Details

**Database Schema:**
```sql
CREATE TABLE redeem_codes (
  code_id UUID PRIMARY KEY,
  code_string VARCHAR(20) UNIQUE,
  value_adc INTEGER,
  created_at TIMESTAMP,
  expires_at TIMESTAMP,
  max_uses INTEGER,
  current_uses INTEGER,
  user_restriction VARCHAR(255), -- email or user_id
  topic_restriction VARCHAR(100), -- subject category
  campaign_id UUID,
  created_by UUID
);

CREATE TABLE code_redemptions (
  redemption_id UUID PRIMARY KEY,
  code_id UUID REFERENCES redeem_codes,
  user_id UUID,
  redeemed_at TIMESTAMP,
  ip_address VARCHAR(45),
  device_fingerprint VARCHAR(255)
);
```

**API Endpoints:**
```typescript
POST /api/codes/generate
// Admin creates codes

POST /api/codes/redeem
// User redeems code

GET /api/codes/validate/:code
// Check if code is valid

GET /api/codes/analytics/:campaign_id
// View campaign performance
```

**Security:**
- Rate limiting (10 attempts per hour)
- Device fingerprinting (prevent abuse)
- IP tracking (detect fraud)
- Unique codes (cryptographically secure)
